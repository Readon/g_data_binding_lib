Content:

Interfaces:

  • BindableCompositeWidget - Specifies interface to access information in composite bindable widgets
  • EditModeControlInterface - Simple edit mode control object for auto widgets in cases when there is no actual data binding or when it needs to be manipulated externally
  • EnumFlagsStringInterface - Provides control interface for enum/flags string representations
  • EnumFlagsValueInterface - Convenience enum flags value interface

Classes:

  • GtkBuildableMapper - Provides easy mapping with composite widgets or layouts loaded from Glade In case of this mapper target should be parent container or composite widget which contains widgets specifying correct Gtk.Buildable names (ID in Glade)
  • GtkBuildableContractMapper - Provides easy mapping with composite widgets or layouts loaded from Glade In case of this mapper target should be parent container or composite widget which contains widgets specifying correct Gtk.Buildable names (ID in Glade)
  • BindingInspector - Data binding inspector window provides runtime information like gtk-inspector does
  • ContractEventDescription
  • EventDescription
  • AutoContainerControl - Controls defaults over how widgets are displayed in auto container
  • AutoContainerModeValues
  • AutoContainerRow - Provides ability to have autofilled widgets which use binding as their central point.
  • AutoContainerValues - Provides simplest modeled container that can be used to easily create and map ListBox rows
  • AutoValueWidget - Provides simplest possible automatic widget which can be created by value type, class type/property name or binding transfer. This is meant to add ability to autocreate row handlers for listbox or other similar needs.
  • AutoValueModeWidget - Provides simplest possible automatic widget which can be created by value type, class type/property name or binding transfer
  • AutoWidget - AutoWidget provides automatic widget creation trough use of DefaultWidgets.
  • CustomCreationDescription - Provides automatic widget creation for either registration or caching purposes. For caching, correct description can be resolved at any time trough DefaultWidgets
  • DefaultWidgets - Registration container for methods that can be used to automate gui creation per object properties or per data types
  • EditModeControl - Simple edit mode control object for auto widgets in cases when there is no actual data binding or when it needs to be manipulated externally
  • EnumFlagsLabel - GtkLabel extension that simply takes advantage of binding to EnumFlagsString and displaying live human readable representation of enum/flags value
  • EnumFlagsMenuButton - EnumFlagsMenuButton provides button with popover that can be used to edit flags and enums
  • EnumFlagsPopover - Provides popover with alread filled up editor for specific enum or flags value type
  • EnumFlagsString - Simple object that can be used for enum/flags string formating
  • EnumFlagsStringControl - Provides common control mechanism for group control of enum/flags representation to string
  • EnumFlagsValues - Provides editable items for EnumClass or FlagsClass in list box as a method of controlling the value.
  • ObjectEventsRow
  • ObjectInspector - Property row for object inspector. This is not to be confused with Gtk.ListBoxRow, this widget only provides contents for that.
  • ObjectInspectorListBoxRow - List box row for ObjectInspectorRow
  • ObjectInspectorRow - Property row for object inspector.
  • Placeholder - Simple placeholder widget that contains icon and description
  • PreflightEventBox - simple preflight event box that manipulates opacity based on where mouse is. When mouse is hovering over event box, full_opacity is used, when not, then inactive_opacity is used
  • PointerInfo - Custom widget to display pointer information for chain display
  • ReferenceMonitorRow - Reference monitor row used in object inspector
  • SimpleEntryBox - Simple popover containing entry with Apply button
  • SizeGroupCollection - Provides collection of named sizegroups. This collection self manages its lifetime based on lifetime of ref_owner specified on creation. Since it has its own memory management only weak references should be ever held
  • SmoothListBoxRow - ListBoxRow extended to support smooth hide/unhide by controlling "revealed" property, which in turn correctly handles "visible" by using timeouts as specified
  • StateImage - Simple Image that has two pixbufs that switch based on value of "state". Once state pixbuf is loaded it is permanently cached and reused per size
  • TitleDescriptionRow - Provides simple ListBox row where title and description are aligned vertically
  • TrackElement - Specifies tracking element that supports connection and logging where events are automatically dispatched to log specified at creation
  • TrackLogElement - Specifies simple log event description to track object events
