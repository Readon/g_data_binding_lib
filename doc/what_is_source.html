What is data binding source?

Back to main page

There are always two sides of data binding that needs to be provided Source and Target. When defining new binding both have to provided in order to enable the connection. While they can be just plain ordinary object, they can as well be data provided by Pointer or Contract. In case of Pointer/Contract data can change without the need to rebind, but this is only possible with Active binding. In this case, Pointer/Contract take care of notifications what needs to be changed and how or even the fact if binding is possible (data they can point could be null or simply object type that does not contain specified property which makes binding impossible)

Binding source explained in simplest way is the main data origin point. Whenever you bind two properties there very slim possibility of every encountering situation when one side would not be prevalent to another

In case of binding, object being edited and widgets displaying/editing it are not of the same importance. Object data is always based on him, while widget could as well display only selected object.

Source is the prevalent side of the binding. Side that generates data persistence
