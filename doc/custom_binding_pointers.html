Pointers have 2 distinct extension possibilities

By changing the way source is resolved

Object? redirect_to (ref redirect_in_play)

allows redirection of source and at the same time breaking the chain. example of this is BindingPointerFromPropertyValue which resolves source it was pointed to initially and then instead of returning that object it resolves value of property that is specified with data_property_name. the fact that it breaks the chain provides with possibility of having source source being pointed out of regular chain

By tapping into data transfer and add custom manipulation (this is only valid for pointers that specify MANUAL as their update type.

signal data_changed (BindingPointer pointer, string data_change_cookie)

doing this allows controlling how data updates will be dispatched and also use it for any other purpose possible. one possible use is signal throttling where there is a need to limit amount of signals per time. example of this is ThrotledManualBindingPointer
