Interfaces:

  • HasDescription - Provides unified point for objects that have description property
  • ObjectInformation - Provides additional way of generalizing information about objects that implement this interface
  • UniqueByProperies - UniqueByProperies is helper interface for ObjectArray and its only purpose is to enable definition of objects which can be compared by properties in uniform fashion
  • BindingContractMapper - Interface that allows easier mapping of object. This is not to be confused with BinderMapper which is very similar. BindingContractMapper does exact same job on contract level which means that binding don't just fall apart when source or target is unvailable.
  • BindingGroup - Interface designed for handling group add/remove of property bindings on contract.
  • BindingInformationInterface - Interface for needed functionality in BindingInformation. This is only intended when there is a need to supply custom binding information and it is also used in BindingGroup
  • BindingStateObjects - IMPORTANT! Best and easiest method to understand state/value objects is to run tutorial demo. Demo focuses on visually exposing all internals and events and makes something complex something really trivial. It will be difference between minutes and god knows how long
  • BindingValueObjects - IMPORTANT! Best and easiest method to understand state/value objects is to run tutorial demo. Demo focuses on visually exposing all internals and events and makes something complex something really trivial. It will be difference between minutes and god knows how long
  • BinderMapper - Interface that allows easier mapping of object. This is not to be confused with BindingContractMapper which is very similar. ContractMapper does exact same job on contract level which means that binding don't just fall apart when source or target is unvailable.
  • BindingDataTransferInterface
  • BindingInterface - Most basic property binding requirement
  • DataFloodDetection - DataFloodDetection interface defines basic flood detection functionality

Classes:

  • Attribute - .Net like attribute object
  • GObjectArray - GObjectArray is holding list of non generic objects and emits signals when objects are added or removed.
  • PropertyInfoAttribute - PropertyInfoAttribute main purpose is to tackle deficiency of ParamSpec not being derived from GObject and as such not really useful for binding
  • RefTimeout - RefTimeout is almost the same as GLib.Timeout with one exception of adding safety layer to its lifetime
  • ReferenceMonitor - Taps into reference lifetime and sends signal when reference is deleted. This monitor object has its own lifetime controlled by object being watched and as such any additional ref/unref is completely unwarranted or unecessary.
  • ReferenceMonitorGroup - Provides grouping of reference monitor object lifetimes in order to centrally manipulate notifications
  • StrictWeakRef - Non-generic class StrictWeakRef is different from WeakRef in two things.
  • TypeInformation
  • WeakRef - .Net C# Non-generic WeakRef like implementation
  • WeakRefWrapper - StrictWeakReference wrapped in GObject
  • BindingPointer - IMPORTANT! Best and easiest method to understand binding pointers is to run tutorial demo. Demo focuses on visually exposing all internals and events and makes something complex something really trivial. It will be difference between minutes and god knows how long
  • BindingContract - IMPORTANT! Best and easiest method to understand binding contracts is to run tutorial demo. Demo focuses on visually exposing all internals and events and makes something complex something really trivial. It will be difference between minutes and god knows how long
  • BindingInformation - BindingInformation holds information about binding as well as controls its creation and unbinding.
  • BindingPointerFromPropertyValue - IMPORTANT! Best and easiest method to understand binding pointers is to run tutorial demo. Demo focuses on visually exposing all internals and events and makes something complex something really trivial. It will be difference between minutes and god knows how long
  • BindingSuspensionGroup
  • BooleanCondition - Used to track property changes and provide their state.
  • ContractStorage - Storage for contracts in order to have guaranteed reference when there is no need for local variable or to have them globally accessible by name
  • CustomBindingSourceState - CustomBindingSourceState is state object implementation. Unlike value objects that are stored as CustomPropertyNotificationBindingSource in order to create flexible implementations state objects can only be stored as CustomBindingSourceState or its subclass. If there is a need for different state object, then classifying it as bool value object might be better
  • CustomBindingSourceValue - CustomBindingSourceValue is one of value objects implementations. It serves as well as example on how to create custom value objects as having usable functionality. Unlike CustomBindingSourceData<T> this one uses GLib.Value for value type
  • CustomPropertyNotificationBindingSource - CustomPropertyNotificationBindingSource is base class for value and state objects.
  • DelayedBindingPointer - DelayedBindingPointer serves throtling data refresh with most basic delay.
  • EnumState - Allows active binding to specific boolean flag and also supports object change if specified object was binding pointer
  • EnumStateGroup - Specifies group monitoring of flags values and provides easy way to add specific monitoring states
  • FlagState - Allows active binding to specific boolean flag and also supports object change if specified object was binding pointer
  • FlagStateGroup - Specifies group monitoring of flags values and provides easy way to add specific monitoring states
  • PointerNamespace - Class serves to store weak references for all pointers and it creates filesystem like structure
  • PointerStorage - Storage for pointers in order to have guaranteed reference when there is no need for local variable or to have them globally accessible by name
  • ProxyProperty - Defines proxy access to one specific property and relays notifications that are connected to it
  • ProxyPropertyGroup - Specifies group monitoring of flags values and provides easy way to add specific monitoring states
  • SignalControlledBindingPointer - BindingPointer subclass that defines data change dispatch is being maintained manually
  • TimerControlledBindingPointer - BindingPointer subclass that defines data change dispatch is being maintained manually and dispatch is being handled by timer
  • Trigger - Simple database like trigger for application purposes. Being that trigger is derived from ProxyPropertyGroup its value gets reevaluated whenever any property in group changes and if conditions are right then trigger is emited.
  • Binder - Binder class is multipurpose class that can be used for different reasons - Controlling property binding creation for either debugging purposes or simply because there is a need for different binding than the one provided by PropertyBinding - It can be interchangeable and specified per contract
  • BindingDataTransfer - Specifies get/set override for property value in binding. Do not store any hard references as it get passed in request and
  • BindingDefaults - Enables classes to specify its own way of property resolving and signaling
  • BindingNamespace - Provides access to all bindings done with any Binder. Manually connected property bindings are not stored
  • GObjectBindingDataTransfer - Specifies get/set override for property value in binding. Do not store any hard references as it get passed in request and
  • GSettingsBindingDataTransfer - Handles transfer between for GSettings objects. If for some reason binding needs to be bound to original source when binding with objects of this type it is possible if binding specifies BindFlags.USE_ORIGINAL_SOURCE_TYPE or BindFlags.USE_ORIGINAL_TARGET_TYPE respectively based on which side of binding it is. In that case data transfer creation is redirected to GObject type of data transfer
  • PropertyAlias - Storage for property aliases for any of possible reasons. - Need for relation to differently named properties in different classes - Simple property registration
  • PropertyBinding - PropertyBinding is reimplementation of GBinding with added functionality. Main purpose for this class is handling data transfer between two properties based on how they were connected
  • EnumFlagsModel
  • EnumFlagsValueObject
  • InvertedListModel - List model that inverts order of displayed items whose lifetime depends solely on lifetime of handled model and as such it shouldn't ever be referenced manually.
  • RowColumn - Base class for row columns
  • RowModel - Row model is base class to generate row representations in list widgets

Enums:

  • DebugMessageType
  • CompareDataBy - CompareDataBy specifies how objects stored in ObjectArray are compared in order to allow handling of unique elements
  • INVALID_ENUM_TYPE - Simple case of invalid enum type
  • BindingPointerUpdateType - Specifies type of update which should be handled on source pointed out by pointer
  • BindingReferenceType - Specifies how BindingPointer should handle reference on data that it points to
  • ContractChangeType - Specifies type of change that happened in contract for binding
  • BindFlags - Flags specified upon creation of new BindingInterface to describe how binding should act
  • BindingSide - Specifies binding side
  • InformationAvailability - Specifies data transfer information availability in order to specify how information is available per specific type registered in BindingDefaults
  • RowOrientation - Specifies orientation how row should treat building when it needs to fill row
