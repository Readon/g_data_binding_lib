MANUAL

Description:

MANUAL

Not everywhere would be suitable just binding on properties and that is when source should specify MANUAL

There are a lot of cases when this kind of binding wouldn't fit the purpose well in which case BindingPointer should be created as MANUAL - it might hammer data too much and cause unnecessary utilization - properties just wouldn't have notifications for specified properties. - binding on signals - binding on timers
