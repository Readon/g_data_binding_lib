Binder

Object Hierarchy:

Description:

public class Binder : Object, HasDescription

Binder class is multipurpose class that can be used for different reasons - Controlling property binding creation for either debugging purposes or simply because there is a need for different binding than the one provided by PropertyBinding - It can be interchangeable and specified per contract

In most all cases Binder is probably not needed. BindingContract uses it internally and by default BindingContract uses get_default() unless specified otherwise. If application never changes that kind of behaviour then tapping into events is trivial as all that application needs is calling set_default(my_binder) that was created with handling that was needed.

The only reason where more than one Binder is needed is when application requires fine grained tapping into events in only selected group of contracts
