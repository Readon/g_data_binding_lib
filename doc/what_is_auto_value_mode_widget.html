What is AutoValueModeWidget and what is its purpose

Back to main page

Most of the functionality and purpose is the same as AutoValueWidget, but there is one big difference

AutoModeValueWidget has more complex setup as it is intended to be used in more complex editors. Most notable differences between AutoValueWidget and AutoValueModeWidget are as following

  • AutoValueWidget only has one state (EDIT or VIEW) and although it can be switched, it is not as effective as AutoValueModeWidget which features both states proactively and when switched between EDIT and VIEW it does that by animation

  • Main intended use of AutoValueModeWidget is to be used inside of AutoContainerRow. When combined with that it makes it perfect auto layouting setup which is exactly how AutoContainerModeValues does its job

  • AutoValueWidget by default unless there is developer intervention loses data being edited when switched between EDIT and VIEW where AutoValueModeWidget does not since it does not need to switch widget completely, but rather just switch internal GtkStack page
