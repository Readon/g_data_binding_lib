What is reverse binding and what is its purpose

Back to main page

While this seems useless in first glance, it is not. At least when using Active binding. When you implement Active contract (group) bindings you often come to the point where one property needs to flow in the other direction which would require creating extra contract. REVERSE_DIRECTION removes that need

Instead of creating workarounds or additional contract, it is much easier if REVERSE_DIRECTION rule is added to the existing one

In case of Passive binding, this probably has no use whatsoever since there it is probably much simpler to just swap arguments and be done with it.
